\index{x86!\Instructions!MOV}
\item[MOV] \RU{загрузить значение}\EN{load value}. \RU{эта инструкция была названа неудачно 
(данные не перемещаются), 
что является результатом путаницы: 
в других архитектурах эта же инструкция называется ``LOAD'' или что-то в этом роде}
\EN{this instruction was named awry resulting confusion (data are not moved), in other architectures 
the same instructions is usually named ``LOAD'' or something like that}.

\RU{Важно: если в 32-битном режиме при помощи MOV записывать младшую 16-байтную часть регистра,
то старшие 16 бит останутся такими же.}
\EN{One important thing: if to set low 16-bit part of 32-bit register in 32-bit mode, high 16 bits
will remain as they were.}
\RU{Но если в 64-битном режиме модифицировать 32-битную часть регистра, то старшие 32 бита обнуляются.}
\EN{But if to modify low 32-bit of register in 64-bit mode, high 32 bits of registers will be cleared.}

\RU{Вероятно, это сделано для упрощения портирования кода под x86-64.}
\EN{Supposedly, it was done for x86-64 code porting simplification.}
